1. ¿Cuál es la diferencia entre Internet y la World Wide Web?

La red de amplitud mundial (World wide web) es un sistema de hipertextos y/o hipermedios enlazados y accesibles a través de Internet. La www es el sistema que usamos para ingresar a Internet, es el medio que nos permite visualizar el contenido de Internet, la red de redes. (Informática, 2010)

2. ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

El protocolo HTTP permite la transferencia de archivos entre un cliente y un servidor web.

Comando Descripción
GET Solicita el recurso ubicado en la URL especificada.
HEAD Solicita el encabezado del recurso ubicado n la URL específica
POST Envía datos al programa ubicado en la URL especificada para que el recurso en esa URI los maneje.
PUT Envía datos a la URL especificada. Pone un recurso en la dirección especificada en la URL. Exactamente en esa dirección. Si no existe, lo crea, si existe lo reemplaza.
DELETE Borra el recurso ubicado en la URL específicada.
PATCH Realizar una actualización parcial de un recurso de OSLC. (no suprime las propiedades de recurso local que no están incluidas en la petición).

(Blogspot, 2013) y (CCM,2018)

5. Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

No, el mensaje del código 200 quiere decir “OK”. Es decir, la solicitud no tuvo problemas se llevo a cabo de forma correcta. (CCM,2018)

6. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

El error 404 indica que no se encontró la página. Eso quiere decir que el servidor no encontró la página solicitada. Así que, a menos que el usuario haya ingresado mal la URL, es error del desarrollador. La causa es que esa pagina se dio de baja del servidor sin dejar alguna para redireccionarla. (CCM, 2018)

7. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

El error 500 indica un error interno del servidor, por eso mismo el usuario no es responsable. La causa puede ser muchísimas razones, una puede ser error en la programación.(CCM, 2018)
