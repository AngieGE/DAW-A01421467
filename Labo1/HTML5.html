URL

Respuesta

Uniform Resource Locator, es el identificador que todo recurso en internet necesita tener. Por ejemplo, páginas de hipertexto, archivos de imagen o sonido. Todas las URL están conformadas por un dominio, el protocolo utilizado para extraer la información (HTTP, HTTPS, FTP, Telnet), tambien se puede especificar un link hacia un subdirectorio. (Indiana University, 2017)

MIME

Respuesta

“Multipurpose Internet Mail Extensions, son una serie de convenciones o especificaciones dirigidas al intercambio a través de Internet de todo tipo de archivos (texto, audio, vídeo, etc.)” (Wikipedia, 2017). Este estándar de internet, también permite recibir caracteres (otros además del ASCII).

XML

Respuesta

Extensible Markup Language salió en 1997. Se ve muy parecido que HTML pero las etiquetas contienen distintas palabras. Esto es porque las etiquetas buscan describir los datos que contienen. De igual forma, la sintaxis es más estricta. XML funciona en cualquier plataforma y ha ganado popularidad desde los 2000s. (Javascript and JQuery, pg.374)

XHTML

Respuesta

EXtensible HyperText Markup Language salió en el 2000, el propósito era introducir un nuevo markup language. Se decidió que HTML4 debía ser reformulado y seguir las reglas de XML, y se llamó XHTML. (Javascript and JQuery, pg.374)

HTML5

Respuesta

La versión más actual de HTML. Se introducen nuevos elementos y atributos. Los nuevos elementos más interesantes de html5 son < header >, < footer >, < article >, < section >, < audio > and < video >. (w3schools, s.f.)
HTML5 es un conjunto de tecnologías. Ya separan la estructura, del estilo, de la funcionalidad. Lo cual nos permite tener mejor manejo de la página.

JS

Respuesta

Con los archivos .js podemos modificar el comportamiento del sitio, al igual que hacerla más interactiva. El archivo Javascript es el ultimo que se añade pues agrega usabilidad a la pagina. JS se escribe en texto plano. (Javascript and JQuery, pg.44,45).

Php

Respuesta

“Php es un lenguaje de código abierto muy popular especialmente adecuado para el desarrollo web y que puede ser incrustado en HTML. Lo que distingue a PHP de algo del lado del cliente como Javascript es que el código es ejecutado en el servidor, generando HTML y enviándolo al cliente.” (Php.net, s.f)

JSP

Respuesta

JSP es muy similar a PHP, sólo que utiliza el lenguaje Java. Con Java Server Pages se pueden crear páginas web dinámicas basadas en HTML y XML. (Wikipedia 2017).

AJAX

Respuesta

Ajax nos permite pedir información al servidor y desplegarla en la página, sin que se tenga que recargar el sitio. Los servidores regresan la información en formato HTML, XML o JSON. Esto nos permite mejorar la experiencia del usuario. (Javascrypt and JQuery, pg368).
