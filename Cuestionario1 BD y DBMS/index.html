Cuestionario 1: BD vs DBMS

Por Angélica Güemes Estrada

1. ¿Qué es entonces una base de datos - DB?

Es un conjunto de archivos relacionados lógicamente. Dentro de estas se encuentran muchos datos los cuales al momento de interpretarlos se vuelven información y nos ayudan a hacer una toma de decisiones.

2. ¿En qué casos es conveniente usar bases de datos?

  • Cuando se tiene una gran cantidad de datos que necesitan ser almacenados en algún lugar.
  • Cuando no sólo se necesitan almacenar sino que consultar ciertos conjuntos de datos para toma de decisiones.
  • Cuando van a seguir recibiendo muchos datos, por ejemplo un distribuidor, que tiene que estar modificando sus datos continuamente.

Algunos lugares interesados en tener BD sería: Fábricas, tiendas, escuelas, distribuidores...

3. ¿Qué es un sistema de gestión de base de datos - DBMS?

“Es un módulo de programa que proporciona el interfaz entre los datos de bajo nivel almacenados en la base de datos y los programas de aplicación y consultas hechos al sistema. ” [1]
Data Base Management System es un software que tiene una colección de datos y se encarga de gestionar estos grandes bloques de información. EL DBMS nos permite acceder a los datos, proporcionando un ambiente óptimo para la gestión, almacenamiento y consulta de estos datos. Algunos ejemplos son: MySQL, SQLServer, Axes (de microsoft).

4. Enlista y explica las funciones/responsabilidades que tienen los DBMS.

  1. Simplificar y facilitar el acceso a los datos.
  2. Tener un tiempo de respuesta mínimo.
  3. Centralizar la información.
  4. “Acceder a los datos por medio de diversos programas de aplicación diferentes que no han sido previamente coordinados”[1] Es el ejemplo del banco, no deben de poder actualizar la BD en el mismo instante.
  5. Segura. Que haya jerarquía, es decir, que no todos los usuarios tengan acceso a todos los datos, sólo a los que necesitan.
  6. Confiable
  7. Traducir sentencias DML a comandos del sistema de archivos de bajo nivel[1].

5. Ejemplifica y justifica en que proyecto de los que has realizado hubiera sido conveniente utilizar una base de datos.

El semestre pasado realicé una APP en Android Studio para una cliente que quería hacer una app prototipo para mostrarla en una junta de negocios. En una parte de la APP se desplegaba un catalogo, en el que el usuario tenía que escoger los muebles de una casa. Primero los sillones, las sillas, las mesas, las camas, y más. Tuvimos que en cada intent poner los muebles que correspondian y hubiera sido mejor tener esa información en una base da datos y simplemente jalar los datos (como leer un archivo). Nos hubiera ahorrado tiempo y además si la app no fuera prototipo se necesitaría agregar la BD, pues los muebles van a ir cambiando con el tiempo, puede eliminar unos o agregar otros.
